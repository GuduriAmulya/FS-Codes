1.1 What is Machine Learning?
Machine Learning is a subset of AI where systems learn from data to make predictions or decisions without being explicitly programmed for each task.
Three core types:

Supervised Learning — learns from labeled data (input → output pairs). Examples: spam detection, image classification.
Unsupervised Learning — finds hidden patterns in unlabeled data. Examples: clustering, dimensionality reduction.
Reinforcement Learning — an agent learns by taking actions and receiving rewards/penalties. Examples: game playing (AlphaGo), robotics.

1.2 The ML Pipeline

 Key ML Concepts You Must Know
Bias-Variance Tradeoff:

High bias = model is too simple → underfits (misses patterns)
High variance = model is too complex → overfits (memorizes noise)
Goal: find the sweet spot. Regularization (L1/L2) helps reduce variance.

Train / Validation / Test Split:

Training set (~70%) → model learns
Validation set (~15%) → hyperparameter tuning
Test set (~15%) → final unbiased evaluation

Loss Functions:

MSE (Mean Squared Error) — regression tasks
Cross-Entropy Loss — classification tasks
Hinge Loss — SVMs

Evaluation Metrics:

Classification: Accuracy, Precision, Recall, F1-Score, AUC-ROC
Regression: MSE, RMSE, MAE, R²

VvisualizeVvisualize show_widget3.2 Attention Mechanism — The Core Innovation
Self-Attention formula:
Attention(Q, K, V) = softmax(QKᵀ / √d_k) × V

Q (Query) — "what am I looking for?"
K (Key) — "what do I contain?"
V (Value) — "what do I actually give?"
√d_k — scaling factor to prevent vanishing gradients

Multi-head attention runs several attention operations in parallel, each learning different relationships (e.g., one head learns syntax, another learns coreference).
python# Simplified attention in PyTorch
import torch
import torch.nn.functional as F

def attention(Q, K, V):
    d_k = Q.size(-1)
    scores = torch.matmul(Q, K.transpose(-2, -1)) / (d_k ** 0.5)
    weights = F.softmax(scores, dim=-1)
    return torch.matmul(weights, V), weights
3.3 Positional Encoding
Transformers have no recurrence (unlike RNNs), so they can't naturally understand word order. Positional encoding adds position information to embeddings:
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
Modern LLMs often use RoPE (Rotary Position Embedding) instead, which handles longer sequences better.

Part 4: LLM Training, Fine-Tuning & Alignment
4.1 Pre-training
LLMs are trained on massive text corpora with next-token prediction (causal LM):
Loss = -Σ log P(token_t | token_1, ..., token_{t-1})
This is called autoregressive language modeling. The model learns by trying to predict what comes next.

pythonfrom langchain_community.document_loaders import PyPDFLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_openai import OpenAIEmbeddings
from langchain_community.vectorstores import FAISS
from langchain.chains import RetrievalQA
from langchain_openai import ChatOpenAI

# 1. Load documents
loader = PyPDFLoader("mybook.pdf")
docs = loader.load()

# 2. Chunk the text
splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks = splitter.split_documents(docs)

# 3. Embed and store
embeddings = OpenAIEmbeddings()
vectorstore = FAISS.from_documents(chunks, embeddings)

# 4. Create retriever + QA chain
retriever = vectorstore.as_retriever(search_kwargs={"k": 3})
qa_chain = RetrievalQA.from_chain_type(
    llm=ChatOpenAI(model="gpt-4o"),
    retriever=retriever
)

# 5. Query
answer = qa_chain.invoke({"query": "What are the main topics covered?"})
print(answer["result"])
5.4 Agents & Tools
An agent uses an LLM as a reasoning engine to decide which tools to call, observe the results, and continue reasoning until it reaches an answer.
The ReAct pattern (Reason + Act):
Thought: I need to find the current weather in Mumbai
Action: search_tool("weather Mumbai today")
Observation: Temperature is 34°C, humid
Thought: I have the answer now
Final Answer: It's 34°C and humid in Mumbai today.
pythonfrom langchain.agents import AgentExecutor, create_react_agent
from langchain_community.tools import DuckDuckGoSearchRun
from langchain_core.prompts import PromptTemplate

# Define tools
search = DuckDuckGoSearchRun()
tools = [search]

# Create agent
agent = create_react_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

# Run
result = agent_executor.invoke({"input": "What's the latest news about GPT-5?"})
5.5 LCEL — LangChain Expression Language
LCEL is LangChain's modern way to compose chains using the | (pipe) operator:
pythonfrom langchain_core.output_parsers import StrOutputParser

# Build a chain: prompt → llm → parse output

# Chains are composable

# Parallel execution

    classification=classifier_chain,
    summary=summarizer_chain

6.4 DSPy
A newer framework that compiles prompts programmatically instead of writing them by hand:
pythonimport dspy

class RAGPipeline(dspy.Module):
    def __init__(self):
        self.retrieve = dspy.Retrieve(k=3)
        self.generate = dspy.ChainOfThought("context, question -> answer")
    
    def forward(self, question):
        context = self.retrieve(question).passages
        return self.generate(context=context, question=question)

7.2 Vector Databases — How They Work
Embeddings convert text into high-dimensional vectors (e.g., 1536 dimensions for OpenAI's text-embedding-3-small). Similar meanings → similar vectors → close in vector space.
Similarity search algorithms:

Cosine similarity — angle between vectors (most common)
Euclidean distance — absolute distance
Dot product — fast, used with normalized vectors

HNSW (Hierarchical Navigable Small World) — the algorithm used by FAISS, Chroma, and Pinecone for approximate nearest-neighbor search at scale.
7.3 Quantization
Reduces model size/memory by using lower-precision numbers:

7.4 Context Window & Token Limits

A token ≈ 0.75 words. "ChatGPT is great" ≈ 4 tokens.
Context window = max tokens the model can process at once
GPT-4o: 128K tokens, Claude: 200K tokens, Gemini 1.5: 1M tokens
Exceeding context = information is truncated → use RAG or summarization chains

7.5 Hallucinations & Mitigation
LLMs can hallucinate — generate confident but wrong information. Mitigations:

RAG (ground answers in retrieved docs)
Lower temperature (more deterministic)
Self-consistency prompting
Output parsing + validation (Pydantic + LangChain)
